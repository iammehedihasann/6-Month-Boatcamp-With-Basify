Setting up Environment for Node.js and Browser Console

Node.js Setup

  1. Download Node.js:Go to the official Node.js and download the latest version suitable for your operating system (Windows, macOS, Linux).
  2. Run the Installer:Once the download is complete, run the installer file. Follow the installation instructions, leaving the default settings unless you need specific changes.
  3. Verify Installation:After installation, open your terminal (Command Prompt on Windows, Terminal on macOS or Linux). Type the command: 
    node -v
    This will display the installed Node.js version, confirming that it is properly set up.
  4. Verify NPM (Node Package Manager):Along with Node.js, npm (the Node Package Manager) is installed automatically. You can check its version by running: 
    npm -v
    5. With these steps, your environment is ready for running JavaScript outside of a browser using Node.js. now i can now start building and running server-side applications or manage packages using npm.

Using Browser Console

  1. Open any modern browser (Chrome, Firefox, Edge).
  2. Right-click on the page and select "Inspect" or press Ctrl + Shift + I (Windows) or Cmd + Option + I (Mac).
  3. Go to the "Console" tab to start writing JavaScript code directly in the browser.
  4. For example, you can write: 
    console.log("Hello, World!");
    and hit Enter to see the output.


1. Variables

Variables are containers for storing data values. Declared using var, let, or const.

let name = "John"; // String
const age = 30; // Number
var isStudent = true; // Boolean
  • let: Allows variable reassignment.
  • const: Constant, cannot be reassigned after its initial assignment.
  • var: Old keyword, has function-scoped behavior.

2. Comments

Comments are ignored by JavaScript and used to explain code or prevent execution.

// This is a single-line comment

/*
This is a multi-line comment
Useful for larger explanations.
*/

3. Data Types

  • String: Text, enclosed in quotes ("Hello", 'World').
  • Number: Integers or decimals (42, 3.14).
  • Boolean: true or false.
  • Null: Represents the intentional absence of any object value (null).
  • Undefined: A variable that has been declared but not assigned a value.
  • Object: Collections of key-value pairs (e.g., { name: "John", age: 30 }).
  • Array: Ordered collections of values (e.g., [1, 2, 3, "apple"]).
